## 卷九:Chrome 组件拆解三连(#98)

接在 #93 之后,把卷九「开源项目阅读」的 Chrome 线补到四个主题([01 OnceCallback → 04 NoDestructor](documents/vol9-open-source-project-learn/chrome/))。新增 02 / 03 / 04,顺带把 01 OnceCallback 的中英文重写了一遍。

每个主题走 pre(打地基)→ full(拆实现)→ hands_on(自己写一遍)的结构,配可编译的 sample 工程。

### 02 WeakPtr(正文 16 篇)

拆的是 Chromium 的 `base::WeakPtr`。注意这套跟标准库 `std::weak_ptr` 不是一回事:std 的版本是控制块外挂,Chromium 这套要被指对象继承 `SupportsWeakPtr`,控制块挂在对象内部,所以能配合 `SequenceChecker` 做线程亲和性检查,也能让 `WeakPtrFactory` 集中作废弱引用,对象析构时一次性失效所有指向它的 WeakPtr,典型做法是把 factory 放成最后一个成员。

- pre 七篇打地基:弱引用与生命周期、侵入式引用计数与 `ScopedRefPtr`、原子操作与内存序、`SequenceChecker` 与 `DCHECK`、concepts / requires 约束、模板 friend 与 `uintptr_t` 技巧、Trivial ABI。
- full 六篇拆主线:动机与 API、控制块骨架、`WeakPtrFactory` 与 last member 惯用法、序列亲和性与懒绑定、Bind 集成、测试与性能。
- hands_on 三篇自己写:设计、实现、测试。

### 03 FlatMap(正文 15 篇)

C++23 给标准库进了 `std::flat_map`(连续存储 + 排序,缓存友好),Chromium 这版是自己撸的,所以能看见从零怎么搭:`FlatTree` 骨架、查找插入的实现、用 `sorted_unique` tag dispatch 避开重复排序、底层搬动时迭代器怎么失效、bulk build 一次构造完。

- pre 六篇:有序关联容器、vector 内部与扩容、复杂度与摊还分析、比较器与透明查找、tag dispatch 与 sorted_unique、EBO 与 pair 存储。
- full 六篇:`FlatTree` 骨架、查找与插入、sorted_unique 构造、迭代器失效与 bulk build、测试与性能。

### 04 NoDestructor(正文 8 篇)

`NoDestructor<T>` 是那个乍看离谱、其实有道理的东西:把析构禁掉,对象一直活到进程结束。动机是绕开全局静态对象退出时的析构顺序坑(顺序未定义,跨翻译单元更是没法控),代价是 LSan 会报泄漏,得诚实交代清楚「这不是真泄漏」。

- pre 两篇:静态存储与初始化、placement new 与 aligned storage。
- full 四篇:动机与 API、核心实现、何时该用、LSan 与泄漏。
- hands_on 两篇:设计与实现、使用与测试。

配套代码都在 [`code/volumn_codes/vol9/full_tutorial_codes/chrome_design/`](code/volumn_codes/vol9/full_tutorial_codes/chrome_design/),本轮新增 12–25 号 sample(`weak_ptr.hpp` / `flat_map.hpp` / `no_destructor.hpp` 几个头文件,加若干驱动 cpp)。
