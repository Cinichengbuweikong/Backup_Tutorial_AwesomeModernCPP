# 分布式桥接

前面的章节里，我们一直在讨论单机上的并发问题：线程、锁、原子操作、无锁数据结构、协程、Actor/Channel。这些是并发编程的地基。但现实中，当一台机器不够用时，你必须面对分布式环境——网络不可靠、时钟不准确、部分失败不可避免。

这一章我们把视角从进程内扩展到网络，看看单机并发的知识如何迁移到分布式场景。我们会讨论分布式系统的根本差异、一致性模型的谱系、共识协议的核心思想，以及 gRPC + C++20 协程在分布式通信中的实践方向。这不是为了让你成为分布式系统专家——而是帮你建立从"单机并发"到"分布式系统"的认知桥梁，知道边界在哪里，旧经验哪些还能用，哪些必须重新思考。
